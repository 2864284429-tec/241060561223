## B. 发现的问题清单

### B-01. HomeController.Index 未读取 Session 用户信息

| 项目 | 内容 |
|------|------|
| **问题描述** | docs/05 §页面卡 1 明确要求"读取当前体验账号信息（Session/Cookie），显示账号名称"，但 HomeController.Index 仅返回 View()，未从 Session 读取用户 |
| **影响** | 首页显示"欢迎回来"但未显示具体用户名（如"欢迎回来，小王"） |
| **严重级别** | 🟡 中 — 功能缺失，但不阻塞 Sprint 1 最低完成线 |
| **关联文档** | docs/05 §页面卡 1、docs/09 §2.3 HomeController |
| **建议** | Sprint 2 实现 Session 登录时一并处理 |

### B-02. 座位列表页已预约卡片可点击进入详情

| 项目 | 内容 |
|------|------|
| **问题描述** | Seat/List.cshtml 中"已预约"座位的卡片仍是 `<a>` 链接（可点击进入详情），原型中已预约卡片应禁用点击（opacity: 0.5 但链接仍生效） |
| **影响** | 用户点击已预约座位会进入详情页，但详情页显示"当前不可预约"，体验略差 |
| **严重级别** | 🟢 低 — 可接受，详情页已有禁用按钮 |
| **建议** | Sprint 2 可优化为 `<div>` 替代 `<a>`，或保持现状 |

### B-03. 座位详情页日期/时段选择器无交互

| 项目 | 内容 |
|------|------|
| **问题描述** | Detail.cshtml 中日期选择器和时段选择器为静态渲染，无 JavaScript 交互，无法选择日期或时段 |
| **影响** | 预约按钮链接到 `#`，无法实际提交预约 |
| **严重级别** | 🟢 低 — docs/13 §7 已记录为 Sprint 2 默认补足项 |
| **建议** | Sprint 2 实现 ReservationController 时一并处理 |

### B-04. 首页"我的预约"链接为空

| 项目 | 内容 |
|------|------|
| **问题描述** | Home/Index.cshtml 中"我的预约"按钮链接到 `href="#"`，无实际路由 |
| **影响** | 点击无反应 |
| **严重级别** | 🟢 低 — Sprint 2 实现 ReservationController.My 时处理 |
| **建议** | Sprint 2 补充路由 |

### B-05. docs/09 中 Device 字段名未完全同步

| 项目 | 内容 |
|------|------|
| **问题描述** | docs/09 §2.4 的 Service 实现要点中仍使用 `Device = seat.Device`，但实际代码已使用 Features。docs/12 记录已修订，但 docs/09 原文未更新 |
| **影响** | 文档与代码不一致，可能误导后续开发者 |
| **严重级别** | 🟡 中 — 文档一致性问题 |
| **建议** | 后续修订 docs/09 时统一为 Features |

### B-06. README 未链接 docs/13

| 项目 | 内容 |
|------|------|
| **问题描述** | README.md "相关文档"列表未包含 docs/13-用户端主链路开发记录.md |
