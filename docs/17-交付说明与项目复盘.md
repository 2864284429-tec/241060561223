## 五、如果再给 1-2 次课，最该补什么

### 优先级 1：预约提交闭环（1 次课）

这是 MVP 的核心缺口。需要完成：
- `ReservationController.Create`（GET + POST）
- 预约提交视图（日期选择器 + 时段列表）
- `ReservationService.CreateReservation`（时段冲突检测）
- 预约成功后跳转"我的预约"

完成后用户端核心链路闭环：选座 → 选时段 → 提交 → 查看。

### 优先级 2：我的预约 + 取消（0.5 次课）

- `ReservationController.My` + 视图
- 取消预约功能（确认弹窗 + 状态更新）

### 优先级 3：统计页（0.5 次课）

- `AdminController.Stats` + 视图
- 座位总数、可用数、预约数统计卡片

### 优先级 4：打磨（如有余力）

- 管理员密码数据库存储
- 预约列表筛选/搜索
- 响应式布局 viewport 测试
