# PRD Lite：图书馆座位预约系统

---

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统（Library Seat Reservation System） |
| 项目类型 | 课程实践项目（模拟真实工作流程） |
| 技术栈 | ASP.NET Core MVC (.NET 8) + Razor 视图 + SQL Server LocalDB + EF Core + Bootstrap |
| 页面总数 | 9 个（用户端 6 个 + 管理端 3 个） |
| 功能总数 | 9 个 MVP 功能 |
| 开发周期 | 2-3 周 |
| 交付方式 | GitHub 或 Gitee 仓库 |

**一句话描述**：让同学们在线上看到哪些座位有空位，提前预约，到了图书馆直接坐下学习。

---

## 2. 项目目标

### 2.1 核心目标

完成一个**可演示、可验收**的图书馆座位预约系统，覆盖"选座 → 预约 → 查看 → 取消"完整链路，以及管理端的座位与预约管理。

### 2.2 验收目标

项目完成时，必须能够演示以下操作：
1. 学生用户浏览座位列表，看到座位状态（可用/已预约）
2. 学生用户选择座位并提交预约
3. 学生用户在"我的预约"中查看预约记录
4. 学生用户取消未到时间的预约
5. 管理员登录管理后台
6. 管理员管理座位信息（增删改）
7. 管理员查看所有预约记录

---

## 3. 目标用户

| 用户角色 | 用户画像 | 使用频率 |
|----------|----------|----------|
| **学生用户** | 大学生，期末复习期间需要长时间在图书馆学习，不想早起排队占座 | 考试季每天 1-2 次 |
| **管理员** | 图书馆值班老师，负责日常座位管理，需要查看预约数据 | 每天查看 1-2 次 |

### 3.1 用户登录方式

| 用户类型 | 登录方式 | 说明 |
|----------|----------|------|
| 学生用户 | 体验账号切换 | 预设 3 个体验账号，通过下拉框或按钮切换 |
| 管理员 | Session/Cookie 登录 | 1 个固定管理员账号，硬编码在代码中 |

## 4. 使用场景

### 4.1 场景 S1：快速预约座位

| 要素 | 内容 |
|------|------|
| 触发条件 | 小王想预约明天上午的座位 |
| 用户行为 | 浏览座位列表 → 选择可用座位 → 选择时间段 → 提交预约 |
| 期望结果 | 预约成功，"我的预约"中显示记录 |
| 异常情况 | 座位已被预约 → 系统提示"该时段已被预约" |

### 4.2 场景 S2：查看预约状态

| 要素 | 内容 |
|------|------|
| 触发条件 | 小王想确认自己的预约 |
| 用户行为 | 进入"我的预约"页面 |
| 期望结果 | 看到预约记录，状态显示"已预约" |

### 4.3 场景 S3：取消预约

| 要素 | 内容 |
|------|------|
| 触发条件 | 小王明天有事去不了 |
| 用户行为 | 在"我的预约"中点击取消 → 确认弹窗 → 确认取消 |
| 期望结果 | 预约取消，座位恢复可用 |
| 异常情况 | 已过时间的预约 → 按钮置灰，提示"已过时，无法取消" |

### 4.4 场景 S4：管理员查看所有预约

| 要素 | 内容 |
|------|------|
| 触发条件 | 李老师想了解今天的预约情况 |
| 用户行为 | 登录管理后台 → 预约管理 |
| 期望结果 | 看到所有用户的预约记录列表 |

### 4.5 场景 S5：管理员管理座位

| 要素 | 内容 |
|------|------|
| 触发条件 | 李老师想新增或修改座位信息 |
| 用户行为 | 座位管理 → 新增/编辑/删除座位 |

## 5. 用户端核心功能

### 5.1 用户首页（U01）

**功能描述**：系统入口页面，展示欢迎信息和快捷导航。

**用户故事**：
> 作为学生用户，我希望打开系统后能看到一个清晰的首页，方便我快速进入座位列表或查看我的预约。

**页面元素**：
- 欢迎信息（显示当前体验账号名称）
- 当前日期显示
- 快捷入口按钮：浏览座位、我的预约
- 今日预约概览（可选）

**最小验收标准**：
- [ ] 页面正常加载，无 JS 错误
- [ ] 显示欢迎信息和当前日期
- [ ] 点击"浏览座位"跳转到座位列表页
- [ ] 点击"我的预约"跳转到我的预约页

---

### 5.2 座位列表（U02）

**功能描述**：展示所有座位及其当前状态，用户可选择可用座位进行预约。

**用户故事**：
> 作为学生用户，我希望看到所有座位的列表，清楚知道哪些座位可用、哪些已被预约，方便我快速选择。

**页面元素**：
- 座位卡片列表（每个卡片显示：座位名称、位置、状态标签）
- 状态标签：绿色"可用"、红色"已预约"
- 点击卡片跳转到座位详情

**最小验收标准**：
- [ ] 页面正常加载，显示所有座位
- [ ] 每个座位显示名称、位置、状态标签
- [ ] 可用座位显示绿色标签，已预约座位显示红色标签
- [ ] 点击座位卡片跳转到详情页

---

### 5.3 座位详情（U03）

**功能描述**：查看单个座位的详细信息，并可发起预约。

**用户故事**：
> 作为学生用户，我希望查看座位的详细信息（位置、容量、设备等），确认后点击预约按钮。

**页面元素**：
- 座位名称、位置、容量、设备信息
- 当前状态显示
- 预约按钮（仅当座位可用时显示）

**最小验收标准**：
- [ ] 页面正常加载，显示座位详细信息
- [ ] 显示座位名称、位置、容量、设备
- [ ] 可用座位显示"预约"按钮
- [ ] 已预约座位不显示"预约"按钮
- [ ] 点击"预约"按钮跳转到预约提交页

---

### 5.4 预约提交（U04）

**功能描述**：选择日期和时间段，提交预约。

**用户故事**：
> 作为学生用户，我希望选择一个日期和时间段，确认后提交预约，系统告诉我是否成功。

**页面元素**：
- 座位信息摘要（名称、位置）
- 日期选择器（默认今天）
- 时间段选择（8:00-21:00，每小时一个时段）
- 确认提交按钮
- 提交成功/失败提示

**业务规则**：
- 时间段粒度：1 小时
- 可选时段：8:00-21:00（共 13 个时段）
- 冲突检测：同一座位同一时段不可重复预约

**最小验收标准**：
- [ ] 页面正常加载，显示座位信息
- [ ] 日期选择器可选择今天及之后的日期
- [ ] 时间段列表显示 8:00-21:00 的 13 个时段
- [ ] 已被预约的时段显示为不可选或灰色
- [ ] 选择日期和时段后点击提交，预约成功并显示提示
- [ ] 尝试预约已占用时段，系统提示"该时段已被预约"

---

### 5.5 我的预约（U05）

**功能描述**：查看当前用户的预约记录列表。

**用户故事**：
> 作为学生用户，我希望看到我的所有预约记录，包括状态和操作按钮。

**页面元素**：
- 预约记录表格（列：座位名称、日期、时间段、状态、操作）
- 状态显示：已预约、已取消、已过时
- 操作按钮：取消（仅对未过时的预约可用）

**最小验收标准**：
- [ ] 页面正常加载，显示当前用户的预约记录
- [ ] 每条记录显示座位名称、日期、时间段、状态
- [ ] 未过时的预约显示"取消"按钮
- [ ] 已过时的预约"取消"按钮置灰
- [ ] 点击"取消"弹出确认框

---

### 5.6 取消预约（U06）

**功能描述**：取消未到时间的预约。

**用户故事**：
> 作为学生用户，我希望取消一个未到时间的预约，系统确认后更新状态。

**页面元素**：
- 预约信息摘要
- 确认取消按钮
- 取消成功提示

**业务规则**：
- 仅允许取消未到开始时间的预约

**最小验收标准**：
- [ ] 点击"取消"弹出确认框，显示预约信息
- [ ] 确认取消后，状态更新为"已取消"

## 8. 核心业务流程

### 8.1 用户预约流程

```
用户首页
  ↓ 点击"浏览座位"
座位列表
  ↓ 选择一个可用座位
座位详情
  ↓ 点击"预约"
预约提交
  ↓ 选择日期和时间段
  ↓ 点击"确认提交"
预约成功提示
  ↓ 自动跳转或手动点击
我的预约（显示新记录）
```

### 8.2 用户取消流程

```
我的预约
  ↓ 找到要取消的记录
  ↓ 点击"取消"
确认弹窗
  ↓ 点击"确认"
取消成功
  ↓ 状态更新为"已取消"
  ↓ 座位恢复为"可用"
我的预约（记录状态已更新）
```

### 8.3 管理员管理流程

```
管理员登录
  ↓ 输入账号密码
  ↓ 登录成功
管理后台首页（预约管理）
  ↓ 点击"座位管理"导航
座位管理页面
  ↓ 新增/编辑/删除座位
  ↓ 操作完成
用户端座位列表同步更新
```

## 11. 核心数据对象

### 11.1 座位（Seat）

| 字段 | 类型 | 说明 |
|------|------|------|
| Id | int | 主键，自增 |
| Name | string | 座位名称（如：A01、B02） |
| Location | string | 位置描述（如：一楼靠窗） |
| Capacity | int | 容量（通常为 1） |
| Device | string | 设备信息（如：有电源、有台灯） |
| IsAvailable | bool | 是否可用（默认 true） |

### 11.2 预约（Reservation）

| 字段 | 类型 | 说明 |
|------|------|------|
| Id | int | 主键，自增 |
| SeatId | int | 外键，关联座位 |
| UserId | string | 用户标识（体验账号 ID） |
| Date | DateTime | 预约日期 |
| TimeSlot | int | 时段（8-20，对应 8:00-20:00） |
| Status | string | 状态：已预约 / 已取消 / 已过时 |
| CreatedAt | DateTime | 创建时间 |

### 11.3 管理员（Admin）

| 字段 | 类型 | 说明 |

## 12. 状态流转说明

### 12.1 预约状态流转

```
[创建预约]
    ↓
  已预约
    ↓
    ├── 用户取消 → 已取消
    ↓
    └── 时间过期 → 已过时
```

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| （无） | 提交预约 | 已预约 | 新建预约记录 |
| 已预约 | 用户点击取消 | 已取消 | 座位恢复可用 |
| 已预约 | 当前时间 > 预约结束时间 | 已过时 | 自动状态（查询时判断） |

### 12.2 座位状态流转

```
[管理员创建座位]
    ↓
  可用
    ↓
    ├── 用户预约 → 已预约（该时段）
    ├── 用户取消 → 可用（该时段）
    ├── 管理员删除 → （记录删除）
    └── 管理员设为不可用 → 不可用
```

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| （无） | 管理员新增 | 可用 | 新建座位记录 |
| 可用 | 用户预约该时段 | 已预约（该时段） | 座位在该时段被占用 |
| 已预约（该时段） | 用户取消 | 可用（该时段） | 座位在该时段恢复可用 |
| 可用 | 管理员删除 | （记录删除） | 座位从系统中移除 |
| 可用 | 管理员设为不可用 | 不可用 | 座位暂时不可预约 |

### 12.3 座位状态判断规则

- 座位在**特定日期+特定时段**的状态是独立的
- 同一座位在不同日期/时段可以有不同的状态
- 列表页显示的状态以**当前日期+当前时段**为基准
