## 本轮选择理由

从 Sprint 3 的 8 张任务卡中，本轮选择 T13-01、T13-02、T13-03 作为核心推进项，理由：

1. **登录是管理端的入口**：没有登录功能，所有管理端页面都无法测试
2. **Session 是权限边界的基础**：未登录跳转是前后端分离的安全底线
3. **三张卡片形成最小闭环**：登录 → 验证 → Session → 跳转，可独立验收
4. **顺带完成 T13-04 和 T13-08 的最小版本**：登录成功后需要一个落脚页（ReservationList），否则用户体验断裂

## 实现详情

### 1. AdminAuthController（T13-01, T13-02, T13-03）

**路由：**
- GET `/Admin/Login` — 显示登录表单
- POST `/Admin/Login` — 提交登录
- POST `/Admin/Logout` — 退出登录

**Session 设计：**
- Key: `IsAdminLoggedIn`，值: `"true"`
- Key: `AdminUsername`，值: 用户名
- 登录成功设置，退出登录清除

**权限守卫：**
```csharp
// 在每个管理端 Controller 的 Action 开头检查
if (!AdminAuthController.IsAdminLoggedIn(HttpContext))
{
    return RedirectToAction("Login", "AdminAuth");
}
```

**错误处理：**
- 用户名/密码为空 → "请输入用户名和密码"
- 凭据错误 → "账号或密码错误"
- 已登录访问 Login → 自动跳转 ReservationList

### 2. AdminHomeController（T13-04）

**路由：**
- GET `/Admin/ReservationList` — 预约管理页

**权限：** 需 Session 验证

**数据来源：** `IReservationService.GetAllBookings()`（Sprint 0 已实现）

**状态显示：** 动态计算"已过时"状态（与用户端一致）

### 3. _AdminLayout.cshtml

**布局结构：**
- 左侧侧边栏（240px，深色背景）
- 侧边栏导航项：预约管理、座位管理、统计（高亮当前页）
- 底部退出登录按钮
- 移动端：汉堡菜单切换侧边栏
- 右侧主内容区

**导航项（与原型一致）：**
- 预约管理 → `/Admin/ReservationList`
- 座位管理 → `/Admin/SeatList`（待实现）
- 统计 → `/Admin/Stats`（待实现）
