# 项目任务板与迭代记录

> 本文件模拟在线任务板（如 GitHub Issues、Trello、Notion），用于跟踪 Sprint 进度。
> 每个 Sprint 完成后更新状态。

---

## 使用说明

### 卡片状态

| 状态 | 含义 |
|------|------|
| ⏳ 待开发 | 任务已创建，尚未开始 |
| 🔨 进行中 | 正在开发 |
| 🔍 待验证 | 开发完成，等待验证 |
| ✅ 已完成 | 验证通过，完成 |
| ❌ 已取消 | 任务取消 |

### 卡片编号规则

`T<Sprint编号>-<序号>`

- Sprint 0 → T10-xx
- Sprint 1 → T11-xx
- Sprint 2 → T12-xx
- Sprint 3 → T13-xx
- Sprint 4 → T14-xx

### 如何使用

1. 开发前：从「待开发」拖到「进行中」
2. 开发中：更新卡片内容（完成百分比、遇到的问题）
3. 开发后：拖到「待验证」，验证通过后拖到「已完成」
