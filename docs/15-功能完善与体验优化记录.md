## 本轮选择理由

从任务板中定位 Sprint 3 剩余未完成任务卡 T13-06、T13-07，选择理由：

1. **T13-06 + T13-07 是管理端最低完成线的阻塞项**：没有座位管理，管理员无法执行核心操作
2. **两张卡片形成最小闭环**：SeatList（列表展示）+ Create/Edit/Delete（CRUD 操作），可独立验收
3. **Service 层已在 Sprint 0 实现**：SeatService.GetAllSeats()、CreateSeat()、UpdateSeat()、DeleteSeat() 均已就绪，本轮只需创建 Controller 和 View
4. **管理端布局 CSS 缺失是隐性阻塞**：_AdminLayout.cshtml 引用了 admin-sidebar/admin-content/admin-mobile-nav 样式类，但 custom.css 中未定义，导致管理端页面布局错乱

| `Controllers/Admin/AdminSeatController.cs` | 新增 | 座位管理控制器：SeatList(GET) + Create/Edit/Delete(POST)，含 Session 守卫 + AntiForgeryToken + 表单校验 |

### 校验变更

| 场景 | 校验规则 | 提示信息 |
|------|----------|----------|
| 新增/编辑座位 | Name 和 Location 非空 | "座位名称和位置不能为空" |
| 新增/编辑座位 | Capacity >= 1 | 自动修正为 1 |
| 删除座位 | 有关联预约时 Service 返回 false | "删除失败，该座位可能有未完成的预约" |

### 提示信息变更

| 场景 | 提示类型 | 内容 |
|------|----------|------|
| 新增成功 | success | "座位新增成功" |
| 新增失败 | error | "座位新增失败，请重试" |
| 编辑成功 | success | "座位信息已更新" |
| 编辑失败 | error | "更新失败，座位可能不存在" |
| 删除成功 | success | "座位已删除" |
| 删除失败 | error | "删除失败，该座位可能有未完成的预约" |
| 校验失败 | error | "座位名称和位置不能为空" |
