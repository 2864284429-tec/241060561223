## 二、Bug 发现与修复记录

### Bug #1：PendingModelChangesWarning（P0 — 已修复）

| 字段 | 内容 |
|------|------|
| **严重程度** | P0 — 应用启动阻断 |
| **现象** | `dotnet run` 启动时抛出 `PendingModelChangesWarning`，EF Core 检测到 HasData 中使用了 `DateTime.UtcNow`，与快照值不一致 |
| **根因** | `AppDbContext.cs` 的 HasData 种子数据中使用了 `DateTime.UtcNow`，每次编译时间不同导致快照不匹配 |
| **修复方式** | 将 `DateTime.UtcNow` 替换为固定值 `new DateTime(2026, 1, 1, 0, 0, 0, DateTimeKind.Utc)` |
| **影响文件** | `DataAccess/AppDbContext.cs` |
| **验证** | `dotnet build` — 0 错误 0 警告；应用正常启动 |

### Bug #2：LocalDB 不可用（P0 — 已修复）

| 字段 | 内容 |
|------|------|
| **严重程度** | P0 — 应用无法运行 |
| **现象** | macOS 环境下无 SQL Server LocalDB，`dotnet run` 因连接 `DefaultConnection` 失败而崩溃 |
| **根因** | 项目原设计依赖 SQL Server LocalDB，开发环境为 macOS 无 LocalDB |
| **修复方式** | 添加 SQLite 回退方案：添加 `Microsoft.EntityFrameworkCore.Sqlite` 包，`appsettings.json` 新增 `SqliteConnection`，`Program.cs` 优先使用 SQLite，`EnsureCreated()` 替代 `Migrate()` |
| **影响文件** | `LibrarySeatReservation.Web.csproj`、`appsettings.json`、`Program.cs` |
| **验证** | 应用启动成功，种子数据自动初始化，8 个座位 + 3 个用户 |

### Bug #3：管理端 500 / 404 路由错误（P0 — 已修复）

| 字段 | 内容 |
|------|------|
| **严重程度** | P0 — 管理端完全不可用 |
| **现象** | `/Admin/Login` 返回 404；`/AdminAuth/Login` 返回 500（`View 'Login' was not found`） |
| **根因** | 控制器类名（`AdminAuthController`、`AdminHomeController`、`AdminSeatController`）与视图文件夹 `/Views/Admin/` 不匹配。ASP.NET Core 约定视图查找路径为 `/Views/{ControllerName}/`，实际视图在 `/Views/Admin/`，导致视图引擎始终找不到文件 |
| **修复方式** | 将 3 个管理端控制器合并为单一 `AdminController`（`Controllers/AdminController.cs`），统一管理登录、预约列表、座位管理。视图位于 `/Views/Admin/`，与控制器名 `Admin` 完全匹配。同时更新所有 Razor 视图中的 `asp-controller` 引用和 `_AdminLayout.cshtml` 中的导航激活状态判断 |
| **影响文件** | `Controllers/AdminController.cs`（新增）、`Controllers/Admin/AdminAuthController.cs`（删除）、`Controllers/Admin/AdminHomeController.cs`（删除）、`Controllers/Admin/AdminSeatController.cs`（删除）、`Views/Admin/Login.cshtml`、`Views/Admin/SeatList.cshtml`、`Views/Shared/_AdminLayout.cshtml` |
| **验证** | 全部路由返回正确状态码；登录→预约列表→座位管理完整流程可走通 |
