# 系统设计说明：图书馆座位预约系统

---

## 1. 系统设计目标

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 可维护 | 代码分层清晰，改一处不影响其他部分 |
| 可扩展 | 新增功能（如用户注册、消息通知）有明确的扩展点 |
| 可演示 | 课堂项目场景，功能完整、流程闭环、页面可用 |
| 低复杂度 | 不引入过度设计，保持实习生可理解和修改 |

### 1.2 设计原则

| 原则 | 具体做法 |
|------|----------|
| 单一职责 | Controller 只做请求分发，Service 只做业务逻辑，DataAccess 只做数据访问 |
| 依赖注入 | 所有 Service 通过 DI 注入 Controller，方便测试和替换 |
| 按功能分包 | 同一功能的 Controller/Service/ViewModel/Entity 放在同一文件夹下 |
| 拒绝 God Class | 单个文件不超过 250 行，超过则拆分 |

## 2. 技术栈说明

### 2.1 核心技术选型

| 技术 | 版本 | 用途 | 说明 |
|------|------|------|------|
| ASP.NET Core MVC | .NET 8 | Web 框架 | 模型-视图-控制器架构 |
| Razor | - | 视图引擎 | 服务端渲染 HTML 页面 |
| Entity Framework Core | 8.x | ORM | 数据库对象关系映射 |
| SQL Server LocalDB | - | 数据库 | 开发环境轻量级数据库 |
| Bootstrap | 5.x | CSS 框架 | 响应式布局和组件 |
| jQuery | 3.x | JS 库 | AJAX 请求、DOM 操作（可选） |

### 2.2 为什么选这些

| 选择 | 理由 |
|------|------|
| MVC 而非 Razor Pages | 课程要求，MVC 更经典，学习价值更高 |
| EF Core 而非 Dapper | 开发效率高，LINQ 查询直观，适合快速原型 |
| LocalDB 而非 SQLite | SQL Server 生态，与生产环境一致 |
| Bootstrap 5 而非 Tailwind | 组件丰富，上手快，适合课程项目 |

### 3.1 目录结构说明

| 文件夹 | 职责 | 关键原则 |
|--------|------|----------|
| Controllers/ | 接收请求、调用 Service、返回视图 | 不写业务逻辑，不直接操作数据库 |
| Services/ | 业务逻辑处理、数据校验、状态流转 | 一个 Service 对应一个业务领域 |
| DataAccess/ | 数据库连接、查询、保存 | 不包含业务判断，只做 CRUD |
| Models/Entities/ | 数据库表结构映射 | 字段与数据库一一对应 |
| Models/ViewModels/ | 页面需要的数据结构 | 一个页面一个 ViewModel（或复用） |
| Views/ | HTML 页面渲染 | 只做数据展示，不写业务逻辑 |

## 4. 分层说明

### 4.1 整体分层架构

```
┌─────────────────────────────────────────────────────────┐
│                      视图层 (Views)                       │
│         Razor 视图，负责 HTML 渲染和用户交互               │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                   控制器层 (Controllers)                   │
│      接收 HTTP 请求，调用 Service，选择视图返回             │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                   业务逻辑层 (Services)                    │
│      处理业务规则、状态流转、数据校验                       │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                 数据访问层 (DataAccess)                    │
│      EF Core DbContext，执行数据库查询和保存               │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                    数据库 (SQL Server LocalDB)            │
└─────────────────────────────────────────────────────────┘
```

### 4.2 各层职责详解

#### Controller 层

| 职责 | 做法 | 反例 |
|------|------|------|
| 接收请求 | 从 URL/表单获取参数 | ❌ 在 Controller 里写 SQL |
| 调用 Service | 通过依赖注入获取 Service 实例 | ❌ 直接 new Service() |
| 选择视图 | 根据 Service 返回结果选择 View 或 Redirect | ❌ 在 Controller 里判断业务状态 |
| 传递数据 | 将 Service 返回的数据放入 ViewModel 传给 View | ❌ 直接传 Entity 给 View |

**Controller 示例骨架：**

```csharp
// SeatController.cs
public class SeatController : Controller
{
    private readonly ISeatService _seatService;

    // 依赖注入
    public SeatController(ISeatService seatService)
    {
        _seatService = seatService;
    }

    // GET: /Seat/List
    public IActionResult List()
    {
        var seats = _seatService.GetAllSeatsWithStatus();
        var viewModel = new SeatListViewModel { Seats = seats };
        return View(viewModel);
    }

    // GET: /Seat/Detail/5
    public IActionResult Detail(int id)
    {
        var seat = _seatService.GetSeatDetail(id);
        if (seat == null) return NotFound();
        return View(seat);
    }
}
```

#### Service 层

| 职责 | 做法 | 反例 |
|------|------|------|
| 业务规则 | 判断预约时间是否合法、是否冲突 | ❌ 在 Controller 里写 if-else |
| 状态流转 | 预约状态从"已预约"变为"已取消" | ❌ 在 View 里直接改数据库 |
| 数据校验 | 检查必填字段、格式合法性 | ❌ 只在前端校验 |
| 调用 DataAccess | 通过接口调用 Repository 或 DbContext | ❌ 在 Service 里写 SQL |

**Service 示例骨架：**

```csharp
// ReservationService.cs
public class ReservationService : IReservationService
{
    private readonly AppDbContext _dbContext;

    public ReservationService(AppDbContext dbContext)
    {
        _dbContext = dbContext;
    }

    // 创建预约
    public ReservationResult CreateReservation(int seatId, string userId, DateTime date, int timeSlot)
    {
        // 1. 检查时段是否已被预约
        var existing = _dbContext.Reservations
            .FirstOrDefault(r => r.SeatId == seatId 
                              && r.Date == date 
                              && r.TimeSlot == timeSlot 
                              && r.Status == "已预约");
        
        if (existing != null)
        {
            return ReservationResult.Fail("该时段已被预约");
        }

        // 2. 创建新预约
        var reservation = new Reservation
        {
            SeatId = seatId,
            UserId = userId,
            Date = date,
            TimeSlot = timeSlot,
            Status = "已预约",
            CreatedAt = DateTime.Now
        };

        _dbContext.Reservations.Add(reservation);
        _dbContext.SaveChanges();

        return ReservationResult.Success(reservation.Id);
    }

    // 取消预约
    public bool CancelReservation(int reservationId, string userId)
    {
        var reservation = _dbContext.Reservations
            .FirstOrDefault(r => r.Id == reservationId && r.UserId == userId);

        if (reservation == null || reservation.Status != "已预约")
            return false;

        // 检查是否已过开始时间
        var reservationStart = reservation.Date.AddHours(reservation.TimeSlot);
        if (DateTime.Now >= reservationStart)
            return false; // 已过时，不能取消

        reservation.Status = "已取消";
        _dbContext.SaveChanges();
        return true;
    }
}
```

#### DataAccess 层

| 职责 | 做法 | 反例 |
|------|------|------|
| 数据库连接 | 通过 DbContext 连接 LocalDB | ❌ 用 SqlConnection 手动连接 |
| 查询数据 | 使用 LINQ 查询 | ❌ 写原生 SQL（除非必要） |
| 保存数据 | 使用 SaveChanges() | ❌ 在 Controller 里调用 SaveChanges |
| 种子数据 | 通过 SeedData 初始化体验账号 | ❌ 手动插入数据库 |

**DataAccess 示例骨架：**

```csharp
// AppDbContext.cs
public class AppDbContext : DbContext
{
    public DbSet<Seat> Seats { get; set; }
    public DbSet<Reservation> Reservations { get; set; }
    public DbSet<DemoUser> DemoUsers { get; set; }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        // 种子数据：3 个体验账号
        modelBuilder.Entity<DemoUser>().HasData(
            new DemoUser { Id = "user1", Name = "小王" },
            new DemoUser { Id = "user2", Name = "小李" },
            new DemoUser { Id = "user3", Name = "小张" }
        );

        // 种子数据：初始座位
        modelBuilder.Entity<Seat>().HasData(
            new Seat { Id = 1, Name = "A-01", Location = "一楼靠窗", Capacity = 1, Device = "有电源", IsAvailable = true },
            new Seat { Id = 2, Name = "A-02", Location = "一楼靠窗", Capacity = 1, Device = "有电源", IsAvailable = true }
            // ... 更多座位
        );
    }
}
```

### 4.3 ViewModel vs Entity 的区别

| 对比项 | Entity（实体） | ViewModel（视图模型） |
|--------|----------------|----------------------|
| 用途 | 映射数据库表结构 | 承载页面需要的数据 |
| 放在哪 | Models/Entities/ | Models/ViewModels/ |
| 字段 | 与数据库一一对应 | 根据页面需求裁剪 |
| 关系 | 包含导航属性 | 可以合并多个 Entity 的数据 |
| 示例 | `Reservation` 包含所有字段 | `MyBookingsViewModel` 只包含列表 + 当前用户信息 |

**为什么需要 ViewModel？**

```csharp
// Entity：包含数据库所有字段

    public int SeatId { get; set; }

    public DateTime Date { get; set; }

public class MyBookingsViewModel

    public string CurrentUserName { get; set; }     // 当前用户名称

{

    public string SeatName { get; set; }    // 关联查询得到，不是 Entity 字段
    public string SeatLocation { get; set; }
    public DateTime Date { get; set; }
    public int TimeSlot { get; set; }
    public string Status { get; set; }
    public bool CanCancel { get; set; }     // 计算属性：是否可取消
}
```

## 8. 异常处理原则

### 8.1 分层异常策略

| 层 | 异常类型 | 处理方式 |
|----|----------|----------|
| Controller | 参数校验失败 | 返回 View + ViewBag.ErrorMessage |
| Controller | 未找到资源 | 返回 NotFound() 或 RedirectToAction |
| Service | 业务规则冲突 | 返回 Result 对象（Success/Fail + 消息） |
| DataAccess | 数据库连接失败 | 向上抛出，由全局异常处理捕获 |
| DataAccess | 查询超时 | 向上抛出，由全局异常处理捕获 |

### 8.2 Service 返回 Result 模式

```csharp
// 通用结果类
public class ServiceResult
{
    public bool IsSuccess { get; set; }
    public string Message { get; set; }
    public object Data { get; set; }

    public static ServiceResult Success(object data = null)
    {
        return new ServiceResult { IsSuccess = true, Data = data };
    }

    public static ServiceResult Fail(string message)
    {
        return new ServiceResult { IsSuccess = false, Message = message };
    }
}

// 预约专用结果
public class ReservationResult
{
    public bool IsSuccess { get; set; }
    public string Message { get; set; }
    public int? ReservationId { get; set; }

    public static ReservationResult Success(int reservationId)
    {
        return new ReservationResult { IsSuccess = true, ReservationId = reservationId };
    }

    public static ReservationResult Fail(string message)
    {
        return new ReservationResult { IsSuccess = false, Message = message };
    }
}
```

### 8.3 Controller 异常处理示例

```csharp
[HttpPost]
public IActionResult Create(int seatId, DateTime date, int timeSlot)
{
    // 获取当前用户（从 Session）
    var userId = HttpContext.Session.GetString("CurrentUserId");
    if (string.IsNullOrEmpty(userId))
    {
        return RedirectToAction("Index", "Home");
    }

    // 调用 Service
    var result = _reservationService.CreateReservation(seatId, userId, date, timeSlot);

    if (result.IsSuccess)
    {
        TempData["SuccessMessage"] = "预约成功！";
        return RedirectToAction("My");
    }

    // 预约失败，返回错误信息
    ViewBag.ErrorMessage = result.Message;
    ViewBag.SeatId = seatId;
    ViewBag.SelectedDate = date;
    return View();
}
```

## 9. Session / Cookie 使用原则

### 9.1 Session 使用场景

| 场景 | Key | 值类型 | 说明 |
|------|-----|--------|------|
| 当前体验用户 | `CurrentUserId` | string | 存储当前选择的体验账号 ID（如 "user1"） |
| 当前用户名称 | `CurrentUserName` | string | 存储当前用户显示名称（如 "小王"） |
| 管理员登录状态 | `IsAdminLoggedIn` | bool | 标识管理员是否已登录 |

### 9.2 Session 配置

```csharp
// Program.cs
builder.Services.AddSession(options =>
{
    options.IdleTimeout = TimeSpan.FromMinutes(30);  // 30 分钟超时
    options.Cookie.HttpOnly = true;                   // 防止 JS 访问
    options.Cookie.IsEssential = true;                // 即使不接受 Cookie 也能用
});

app.UseSession();  // 启用 Session 中间件
```

### 9.3 Session 读写示例

```csharp
// 设置 Session（切换体验账号时）
HttpContext.Session.SetString("CurrentUserId", "user1");
HttpContext.Session.SetString("CurrentUserName", "小王");

// 读取 Session
var userId = HttpContext.Session.GetString("CurrentUserId");
var userName = HttpContext.Session.GetString("CurrentUserName");

// 管理员登录
HttpContext.Session.SetString("IsAdminLoggedIn", "true");

// 检查管理员是否登录
var isAdmin = HttpContext.Session.GetString("IsAdminLoggedIn") == "true";
if (!isAdmin)
{
    return RedirectToAction("Login", "AdminAuth");
}
```

### 9.4 Cookie 使用场景

| 场景 | 说明 | 建议 |
|------|------|------|
| 记住体验账号 | 用户关闭浏览器后记住上次选择的账号 | 可选，用 Cookie 存储 |
| 主题偏好 | 深色/浅色模式 | 本期不做 |
| 语言偏好 | 中文/英文 | 本期不做 |

### 9.5 Session vs Cookie 选择

| 对比项 | Session | Cookie |
|--------|---------|--------|
| 存储位置 | 服务端（内存/Redis） | 客户端（浏览器） |
| 安全性 | 高 | 低（可被篡改） |
| 容量 | 无限制 | 4KB |
| 生命周期 | 浏览器关闭或超时 | 可设置过期时间 |
| 用途 | 登录状态、敏感信息 | 偏好设置、非敏感信息 |

**本期项目建议**：只用 Session，不用 Cookie。体验账号切换用 Session 足够，30 分钟超时符合课堂演示场景。
