## C. 必改项

### C1：目录结构冗余

**问题**：同时存在 `Data/` 和 `DataAccess/` 两个文件夹，`Data/` 下有 `LibrarySeatContext.cs`，`DataAccess/` 下有 `AppDbContext.cs`，职责重叠。

**修复方案**：

- 删除 `Data/` 文件夹
- 所有数据库相关内容统一放在 `DataAccess/`
- 目录结构改为：

```
DataAccess/
├── AppDbContext.cs
├── SeedData.cs
└── (可选) Repositories/
```

---

### C2：类型命名不一致

**问题**：

| Service 接口返回类型 | ViewModel 类型 | 问题 |
|----------------------|----------------|------|
| `SeatListItem` | `AdminSeatItem` | 同为座位列表项，命名不统一 |
| `BookingItem` | `BookingItemViewModel` | 同为预约记录项，命名不统一 |

**修复方案**：

统一命名规则：

| 场景 | 类型名称 | 放置位置 |
|------|----------|----------|
| 座位列表项（用户端） | `SeatListItem` | Models/ViewModels/ |
| 座位列表项（管理端） | `AdminSeatListItem` | Models/ViewModels/Admin/ |
| 预约记录项（用户端） | `BookingItem` | Models/ViewModels/ |
| 预约记录项（管理端） | `AdminBookingItem` | Models/ViewModels/Admin/ |
| 座位详情 | `SeatDetailViewModel` | Models/ViewModels/ |

Service 接口返回类型应与 ViewModel 类型保持一致，或明确说明 Service 返回 Entity、Controller 负责转换为 ViewModel。

---

### C3：Controller 错误处理方式矛盾

**问题**：第 8.3 节示例使用 `ViewBag.ErrorMessage` 传递错误信息，但文档第 4.2 节主张"将 Service 返回的数据放入 ViewModel 传给 View"。

**修复方案**：

方式一：使用 TempData（推荐，简单）

```csharp
if (!result.IsSuccess)
{
    TempData["ErrorMessage"] = result.Message;
    return RedirectToAction("Create", new { seatId });
}
```

方式二：使用错误 ViewModel

```csharp
if (!result.IsSuccess)
{
    var viewModel = new ReservationCreateViewModel
    {
        SeatId = seatId,
        ErrorMessage = result.Message
    };
    return View(viewModel);
}
```

选择一种并统一全文。

## D. 建议优化项

### S1：AdminAuthService.cs 缩进错误

第 69-70 行：

```markdown
│   └── IAdminAuthService.cs        # 管理员认证接口
│       AdminAuthService.cs         # 管理员认证实现
```

应改为：

```markdown
│   ├── IAdminAuthService.cs        # 管理员认证接口
│   └── AdminAuthService.cs         # 管理员认证实现
```

---

### S2：Result 类简化

当前定义了两个 Result 类：

- `ServiceResult`（通用，含 `object Data`）
- `ReservationResult`（专用，含 `int? ReservationId`）

**问题**：`ServiceResult` 未被任何示例使用，且 `object Data` 类型不安全。

**建议**：

- 删除 `ServiceResult`
- 保留 `ReservationResult` 作为预约专用
- 其他操作（如取消、删除）统一返回 `bool` 或新建简单的 `OperationResult`

---

### S3：CancelReservation 返回类型

**当前**：返回 `bool`，无法携带错误消息。

**建议**：改为返回 `ServiceResult`：

```csharp
ServiceResult CancelReservation(int reservationId, string userId);
```

调用方可根据 `IsSuccess` 和 `Message` 显示不同提示。

---
